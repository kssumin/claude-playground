## EXPLAIN

### 토스의 핵심 테스트 철학

모든 코드에 테스트를 작성하는 것은 비효율적이다.
토스는 **"작성 가치를 고려하여 선택적으로 작성"** 하는 실용적 접근을 채택했다.

> 핵심 원칙: 비즈니스 가치가 높은 20%의 테스트로 80% 이상의 신뢰성을 확보한다.

### 테스트를 쓰지 않는 기준

- 이벤트성 기능 (단기 수명 코드)
- 변경 빈도가 낮고 장애 위험도가 낮은 기능
- 수동 테스트로 충분히 검증 가능한 단순 로직

### 테스트를 반드시 쓰는 기준

- **핵심 비즈니스 로직** (결제, 정산, 알람 발송 등)
- **장애 발생 시 피해가 큰 기능**
- **변경이 잦고 회귀 위험이 있는 로직**

## QUIZ

**Q1.** 토스가 "테스트를 선택적으로 작성"하는 이유는 무엇인가?

A) 테스트 작성이 어려워서
B) 모든 테스트 작성은 비효율적이며, 비즈니스 가치 중심으로 선별하는 것이 더 효과적
C) 테스트 도구가 부족해서
D) 팀 규모가 작아서

**정답: B**
