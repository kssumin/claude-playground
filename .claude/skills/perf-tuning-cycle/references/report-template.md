> **논리 완결성 — 분석 작성 후 아래 4가지 확인**
> 1. **이론값 사용 시**: "이론 상한" 표기 필수. avg 기반 계산은 tail 분포가 경쟁 유발하면 실제 한계가 낮음을 1줄 추가
> 2. **자원 대비 효과 gap**: 투입(N배) vs 결과(M배) 불일치 시 → 구체적 병목 원인 특정 (외부 호출 응답시간인지, 스레드 경쟁인지 등)
> 3. **수치 출처**: 실패율·지연율·baseline 수치는 설정값인지 실측값인지 괄호로 명시
> 4. **기각 논리**: 특정 옵션 기각 시 → 독자가 제기할 수 있는 대안("그러면 X는?")까지 선제적으로 답변
