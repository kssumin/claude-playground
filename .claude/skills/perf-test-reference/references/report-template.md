# 결과 보고서 템플릿

아래 템플릿을 매 라운드마다 작성한다.

````markdown
# Round {N}: {라운드 제목}

| 항목 | Before | After | 변경 이유 |
|------|--------|-------|----------|
| {설정명} | {이전 값} | {변경 값} | {왜 이 값으로 바꿨는지} |

**변경한 것**: {정확히 무엇을 바꿨는지 1줄 요약}
**변경하지 않은 것**: {나머지 조건이 동일함을 명시}

## 테스트 조건

| 항목 | 값 |
|------|-----|
| k6 스크립트 | `scenarios/load-test.js` (커밋: {hash}) |
| 부하 프로파일 | write {N} req/s, read {N} req/s, {M}분 |
| 초기 데이터 | TRUNCATE + FLUSH 완료 |
| JVM warm-up | dry-run 30초 완료 |
| 테스트 시각 | {YYYY-MM-DD HH:MM} ~ {HH:MM} |
````

## 개선 효과 판정 기준

| 비교 항목 | 유효한 비교 | 무효한 비교 |
|----------|------------|------------|
| 동일 k6 스크립트 | ✅ | |
| 동일 데이터 초기 상태 | ✅ | |
| 1개 변수만 변경 | ✅ | |
| k6 스크립트 변경 + 설정 변경 | | ❌ before/after 비교 불가 |
| 2개 이상 동시 변경 | | ❌ 기여도 분리 불가 |
