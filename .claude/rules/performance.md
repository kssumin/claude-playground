# 성능 최적화

## 모델 선택 전략

**Haiku 4.5** (Sonnet 90% 성능, 3배 비용 절감):
- 빈번한 호출의 경량 에이전트
- 페어 프로그래밍 및 코드 생성

**Sonnet 4.5** (최고 코딩 모델):
- 주요 개발 작업
- 멀티 에이전트 워크플로우 오케스트레이션

**Opus 4.5** (가장 깊은 추론):
- 복잡한 아키텍처 결정
- 최대 추론이 필요한 작업

## 컨텍스트 윈도우 관리

컨텍스트 윈도우 마지막 20%에서 피할 작업:
- 대규모 리팩토링
- 여러 파일에 걸친 기능 구현
- 복잡한 상호작용 디버깅

낮은 컨텍스트 민감도 작업:
- 단일 파일 수정
- 독립 유틸리티 생성
- 문서 업데이트
- 단순 버그 수정

## 빌드 트러블슈팅

빌드 실패 시:
1. **build-error-resolver** agent 사용
2. 에러 메시지 분석
3. 점진적 수정
4. 각 수정 후 검증
