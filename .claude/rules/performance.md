# 성능 최적화

## 모델 선택
- Haiku: 경량 에이전트, 페어 프로그래밍. Sonnet: 주요 개발. Opus: 복잡한 아키텍처 결정

## 컨텍스트 윈도우
- MUST: 마지막 20%에서 대규모 리팩토링, 다파일 구현, 복잡한 디버깅 금지
- SHOULD: 단일 파일 수정, 유틸리티 생성, 문서 업데이트는 저컨텍스트 작업

## 빌드 트러블슈팅
- MUST: 빌드 실패 시 build-error-resolver agent 사용 → 에러 분석 → 점진적 수정

## 상세 가이드
모델별 상세, 컨텍스트 전략 상세, 빌드 워크플로우는 `performance-reference` 스킬을 참조하라.
