# Coding Style

## 불변성 (CRITICAL)
- MUST: 새 객체 생성 (data class copy()), 입력 파라미터 직접 변경 금지
- MUST: val 우선, var는 반드시 필요한 경우만. 불변 컬렉션 사용

## 파일 구성
- MUST: 200-400줄 기본, 최대 800줄. 기능/도메인별 정리
- MUST: 높은 응집도, 낮은 결합도

## 에러 처리
- MUST: 의미 있는 컨텍스트와 함께 처리, 사용자 친화적 메시지
- NEVER: 예외를 조용히 삼키지 않음

## 인프라 설정 명시
- MUST: 운영 중요 설정은 기본값이라도 명시적 선언 (코드 = 문서, 의도 가시성)

## 입력 유효성 검증
- MUST: 시스템 경계에서 모든 외부 입력 검증 (Bean Validation, require/check)

## 수술적 변경
- MUST: 작업에 필요한 부분만 수정, 무관한 개선 금지
- MUST: 기존 스타일 유지. 고아 코드만 제거, 기존 데드 코드는 요청 없이 제거하지 않음

## 상세 가이드
Kotlin 스타일, 코드 품질 체크리스트는 `coding-style-reference` 스킬을 참조하라.
